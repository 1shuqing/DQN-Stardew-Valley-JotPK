# 用DQN训练神经网络玩草原之王
注意：目前的模型文件只适用于草原之王第一小关，因为训练代码中没有做通关后的条件判断（方便做第一关的测试训练）
### 星露谷要达成全成就居然要将这个小游戏一命通关，自己打不过于是尝试用模型解决
  这是一个将yolo检测结果用于判断游戏状态与奖惩的尝试，最初没有考虑到这样一个按人类游玩体验来说操作和通关条件很简单的小游戏，居然难以用基础的图像检测写出效果好且逻辑简单的游戏状态判断方法。可以说用上yolo训练一个能识别画面中敌人和玩家的模型用于获取必须的游戏状态是迫不得已了

将如图的检测结果包含的信息转化为用于判断状态的信息（玩家存活、敌人数量、杀敌、通关）就简单多了
  ![image0](https://github.com/user-attachments/assets/0d80529d-19fe-4718-8c77-5a703b940b1b)

只针对第一关出现的怪物和玩家训练了yolo模型（不包括道具）

游戏模型训练在代码默认配置条件下300轮左右会过拟合，目前训练的最优游戏模型是best_model


## 如何使用
1、`下载best.pt、best_model和DQN_play.py`

2、`草原之王打开后拖到屏幕左上角默认尺寸（默认设置，可以在代码里更改，不建议，因为检测时间剩余条的尺寸也要改）`

3、`py文件改好两个模型路径后，运行`

### 如果你想复现训练或者尝试后续关卡的训练
  将本仓库下载后运行`DQN_2.py`,`DQN_NET.py`是将yolo检测到的坐标信息与剩余时间信息归一化后拼接到截取的游戏画面resize好的张量后一同作为神经网络的输入的版本，也能训练，但训练更难拟合，个人认为输入信息过于复杂反而效果没那么好（也可能是有关键的东西没有掌握，导致状态判断的结果与神经网络的计算结果相违背）。

  除此之外，你可以尝试将通过每一关后的处理逻辑完善，让后续的关卡也能参与（目前是通关后处理仅是进入下一轮训练），`或许每个关卡单独训练一个模型是更优解？毕竟后关的训练次数是更难得的，单个模型或许在第一关就训练的过拟合了还测试不到几次第二关`,必须要注意的是用于检测游戏信息的yolo模型并不支持所有怪物的识别，如有需要我可以额外上传图片数据，自己打标后训练即可。
